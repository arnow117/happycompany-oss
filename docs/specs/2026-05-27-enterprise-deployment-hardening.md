# 企业部署加固

**日期**: 2026-05-27
**状态**: 已采纳

## 背景

HappyCompany 从单企业 demo 进入多企业平台部署阶段。企业部署需要保证控制台 API 默认受保护、租户参数不会误回退到真实客户、人员绑定只能指向本企业数字员工，并且生产环境不会在无管理员口令或明文凭证状态下运行。

## 行为要求

- 生产环境必须配置管理员 Token；未配置时服务拒绝启动。
- `/api/admin/session` 在受保护模式下校验 Bearer Token；错误或缺失 Token 返回 401。
- 除健康检查、登录校验和首次 setup 外，`/api/*` 默认要求管理员 Token。
- 企业租户 ID 必须显式且合法；非法租户返回 400，不静默回退到 `acme`。
- 企业人员绑定的 `entryEmployee` 与 `visibleEmployees` 必须存在于当前企业数字员工集合。
- 数字员工注册以租户为作用域，不允许同名员工跨企业互相覆盖。
- 工作流、人员、数字员工页面使用全局当前企业上下文请求数据。

## 部署要求

- `config.json` 使用 `$HAPPYCOMPANY_ADMIN_TOKEN` 等环境变量占位。
- 企业凭证不得提交明文值；生产部署用环境变量或加密配置。
- 运维文档必须说明生产必填环境变量、数据目录备份范围和验证命令。

## 验收

- 未认证请求访问企业人员、租户、模板、聊天历史等控制台数据时返回 401。
- 传入非法 tenant 时返回 400，不读写 `corp/acme`。
- 绑定跨租户或不存在的数字员工时返回 400。
- 两个租户拥有同名数字员工时，注册表中保留两个实例。
- 完整前端 build 和用户故事 E2E 通过。
